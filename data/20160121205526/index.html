クラスとインスタンスとは

そもそもクラスとインスタンスとはどのようなものでしょうか。具体的な例で考えてみます。以下の図をみてください。

クラスとは簡単に言ってしまえばひな形となるものです。車で言えば車の設計図ですね。インスタンスとはクラスを元にして作られるデータのことです。上の例で言えば設計図から作られて実際に販売されている車のことです。

要点チェック

クラスはデータを作るためのひな形

いろんな種類の車が世の中には販売されていますがすべての車で共通する部分があります。「車体、ハンドル、タイヤ」を持っていることや「アクセルで進む、ブレーキで止まる」などです。クラスではこれらの共通している属性を変数やメソッドとして定義することができます。

  • 属性(変数名)

    • 車体
    • ハンドル
    • タイヤ

  • メソッド

    • アクセルで進む
    • ブレーキで止まる

要点チェック

インスタンスはクラス(ひな形)から作られるデータ

インスタンスはクラスから作られるデータです。インスタンスはクラスの特徴を引き継ぐので車の設計図(クラス)から作られたベンツやフェラーリなどの車(インスタンス)も「車体、ハンドル、タイヤ」があり、「アクセルで進む、ブレーキで止まる」という特徴を持ちます。

しかし、「車体、ハンドル、ブレーキ」は車の種類によって異なります。すなわち、インスタンスによって詳細は異なるわけです。

変数名 自動車1 自動車2
車体 車体1 車体2
ハンドル ハンドル1 ハンドル2
タイヤ タイヤ1 タイヤ2


「車体、ハンドル、タイヤ」を持つということは全車で共通なのでクラスで定義し、ハンドルの種類などは個々の車で異なるのでインスタンスごとに定義します。

要点チェック

実は使っていたクラスとインスタンス

このクラスとインスタンスですが実は出てきたのは初めてではありません。今まで使っていた"Hello World"のような文字列はStringクラスのインスタンスです。[]や["いぬ", "ねこ"]のような配列はArrayクラスのインスタンスです。

  string2 = "こんにちわ　Ruby"  # インスタンス

string1とstring2は文字列のクラスであるStringクラスのインスタンスです。同じクラスなので文字を持つということは共通しています。さらに共通のメソッドを持ちます。

クラスにメソッドを定義しよう

クラスには属性と処理を定義できます。まずは処理(メソッド)を定義してみましょう。
クラスに定義できるメソッドにはクラスメソッドとインスタンスメソッドの2種類あります。

クラスメソッド

クラスが使用できるメソッドです。クラスメソッドを定義したクラス自身が使えます。

インスタンスメソッド

インスタンスが使用できるメソッドです。インスタンスメソッドを定義したクラスのインスタンスが使えます。クラス自身は使えません。

    end

クラスメソッドとインスタンスメソッドの違い

ここでクラスメソッドとインスタンスメソッドの違いについておさらいしましょう。

書き方の面だけみれば、両方ともクラスの定義内に書きますが、クラスメソッドはdef self.classMethodのようにself.をメソッド名の前につけ、インスタンスメソッドdef instanceMethodのようにself.をつけません。

【例】

  • クラスメソッドの定義
1
2
3
4
5
 
  class Class
    def self.classMethod
      # クラスメソッド
    end
  end
  • インスタンスメソッドの定義
1
2
3
4
5
 
  class Class
    def instanceMethod
      # インスタンスメソッド
    end
  end

では、どんなときにクラスメソッドを使い、どんなときにインスタンスメソッドを使うのでしょうか。

そのクラスで共通して行う、またはクラス全体に対して何かしらの操作を行う、かつインスタンスの属性に関係のない処理をクラスメソッドとして定義します。ここで重要なことはインスタンスに関係ないということです。
逆に、インスタンスの属性を用いる処理はインスタンスメソッドとして定義します。

  • インスタンスメソッドとクラスメソッドの違い
特徴 インスタンスメソッド クラスメソッド
定義方法 メソッド名の前にselfはつけない メソッド名の前にselfをつける
用途 インスタンスごとの属性を用いる場合 属性は関係のない共通の処理
呼び出せるオブジェクト クラスのインスタンス クラス自身


具体的な例を見てみましょう。以下のように定義されたHumanクラスがあるとします。

【例】

「世界中の人口を表示する」メソッドは人というクラス全体に関するメソッドで、しかもそれぞれ個々のインスタンス(一人一人)に関係がないので、クラスメソッドとなります。

【例】

クラスに変数を定義しよう

クラスには属性を定義できます。属性は変数という形で定義されます
クラスに定義できる変数にはクラス変数とインスタンス変数の2種類あります。

クラス変数

クラス全体で使用できる変数です。クラス内であればどこでも使えるのでクラスメソッド、インスタンスメソッドの両方で使うことが出来ます。クラスで使ってもインスタンスで使っても値は同じです。

インスタンス変数

インスタンスが使用できる変数です。インスタンスでしか使用できないのでインスタンスごとにこの変数の値は異なります。

Rubyで最初から使える代表的なクラス

クラスは自分で定義できるものの他に、Rubyに最初から定義されているクラスがいくつかあります。その中でも代表的なクラスであるStringクラスとArrayクラスを使ってみましょう。

Rubyの文字列と配列はnewメソッドを実行させなくてもオブジェクトのインスタンスを生成することができる仕様になっています。

Stringクラス

Stringクラスは文字を扱うためのクラスです。いままで使っていた文字列オブジェクトはStringクラスのインスタンスです。

1
2
 
  string1 = "Hello World" # Stringクラスのインスタンス
  string2 = 'Hello Ruby'  # Stringクラスのインスタンス

要点チェック

String.newを使おう

今まで、文字列を使う際シングルクォーテーション もしくは ダブルクォーテーションで囲んであげていたと思いますが、
実はこれがStringクラスのインスタンスを生成する書き方なのです。

クォーテーションで囲んだものは全てStringのインスタンスとなります。

1
 
  test = 'これはStringのインスタンスです'

つまり、プログラム内部ではString.newをしているのです。

1
 
  test = String.new('これはStringのインスタンスです')

String.newで書いてもクォーテーションで囲っても出来るインスタンスは全く同じなので、
書きやすいクォーテーションで書くのが一般的です。

確認として、本当にStringクラスのインスタンスなのかどうか確かめてみましょう。

クラスのインスタンスかどうかはinstance_of?(クラス名)メソッドで確認することができます。
trueが返ってきたらインスタンス、falseが返ってきたらインスタンスではないということがわかります。

1
2
 
  test = 'これはStringのインスタンスです'
  test.instance_of?(String) # true

また、ダブルクォーテーション で囲んでも同等です。

1
2
 
  test = "これはString.newと同じです"
  test.instance_of?(String) # true

どちらもStringクラスのインスタンスだということがわかります。

インスタンスということは、当然インスタンスメソッドが使えます。
今までに出てきたと思いますが、文字数を数えるためのlengthメソッドも
インスタンスメソッドの一つで、インスタンスに対して実行させているメソッドです。

要点チェック

Array.newを使おう

String.newと同様なものにもう一つArray.newがあります。

1

1

このように、 Array.newで配列のインスタンスを作ることができ、配列に対してインスタンスメソッドが使えるようになります。

クラスの継承

複数のクラスに同じメソッドを定義したい場合、それぞれのクラスに全てのメソッドを定義すると、同じ記述を何度も繰り返す事になってしまいます。結果コード量が多くなり、管理も難しくなります。これを防ぐために「クラスの継承」について学んでいきましょう。

クラスの継承

あるクラスに定義されたメソッドを、別のクラスで利用出来るようにすることを継承と言います。

クラスの継承には、親クラスと子クラスの関係があります。元となるクラスを親クラス、親クラスのメソッドを引き継ぎ新しく作成するクラスを子クラス と呼びます。


では具体的な例を見てみましょう。

ここでは、2つのクラスを作成したいとします。パトカーのようなインスタンスを生成するクラスとトラックのようなインスタンスを生成するクラスです。どちらも共通して車としての特徴を持っているので、共通する「車としての特徴」を親クラスCarの中でまとめ、パトカーとトラックそれぞれの特徴はそれぞれのクラスPatrolCar, TruckCarでのみ定義することとします。

車の特徴(親クラス)

  • 加速する
  • 減速する
  • ホーンを鳴らす

パトカーだけの特徴(子クラス)

  • サイレンを鳴らす

トラックだけの特徴(子クラス)

  • 荷物を運ぶ

車の特徴(親クラス)を継承する事で、パトカー(子クラス)とトラック(子クラス)に車の特徴を記述する必要がなくなります。結果、パトカーとトラックだけの特徴が分かりやすくなっています。
