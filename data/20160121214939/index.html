学習の目標

エラーが発生した際自力で原因を突き止め、正常にプログラムが作動するよう修正できるようになりましょう。

エラー解決

プログラミングをする上でエラーは必ず起こります。迅速に修正するためには、エラー解決の方法を知っておく必要があります。

本章ではエラー解決の手法を学んだ後に、実際にエラーを起こすファイルを実行し、それを解決することでエラー解決の力を身に付けましょう。

まずはエラー文を読みましょう。エラー文とは、あるファイルが正常に実行されなかった際の原因を報告してくれる文です。多くの場合、エラー文はターミナル上でファイルを実行した直後の行に出力されます。

今回の場合のエラー文は以下の部分です。

1
2
 
  /Users/koichiroabe/error_question/error_sample.rb:17: syntax error, unexpected end-of-input, expecting keyword_end
  # エラー文

エラー文には以下の2つの重要な情報が含まれています。

  1. どんな種類のエラーか

  2. ファイルの何行目でエラーが起きたか

1.に関して

今回は以下の部分がエラーの種類を示しています。

syntax error, unexpected end-of-input, expecting keyword_end

syntax errorは文法に関するエラーだということです。unexpected end-of-input, expecting keyword_endは、構文を閉じるための<end>の数がおかしいことを示しています。

2.に関して

Rubyは、エラー文においてエラーが起こっている場所をちゃんと教えてくれます。

error_sample.rb:17:、つまり、error_sample.rbの17行目でエラーが発生しているよということです。しかし、今回のエラーは<end>の数によるものです。このエラーの場合、該当の行でエラーが起きているわけではありません。Rubyが、endの必要な部分までコードを読んで、そこでエラーを検知するからです。ほとんどの場合はエラーが起こっている場所を正確に示しますが、そうでない場合もあるので注意しましょう。

以上の情報からわかることは、このファイルのどこかに<end>の足りない構文があるということです。早速チェックしてみましょう。

次にエラーが起きた原因ですが、wrong number of arguments (1 for 0)とあります。

wrong number of arguments

wrong number of arguments とは、引数の数が違う、いう意味です。(1 for 0)と後ろに書かれている場合、メソッドを呼ぶ際には1つの引数を渡しているのに、メソッドを定義している箇所では何も受け取っていないよ、という意味になります。

とりあえずエラー箇所である13行目を見てみると、一見普通にクラス定義がされているようです。

しかし、よく見ると間違いがあるのに気が付きます。呼び出す箇所(10行目)では引数を書いているのに、13行目の関数定義には引数を受け取る記述がありません。
